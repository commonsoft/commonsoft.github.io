Hexo 기본 셋팅을 끝낸 뒤 webstorm 혹은 다른 에디터를 활용하여 블로그 글 작성한다.

사전작업

github에 commonsoft의 hexoblog를 clone한다.

만약 해당 repo가 보이지 않을시 담당자에게 문의한다.

기본적인 clone 형태는 .deploy_git하고 public이 없는 형태이다.

해당되는 폴더가 있을 경우 다음을 실행한다.

Content 작성

Content 작성은 아래의 명령어를 커맨드로 입력하면 source/_posts 에 파일이 생성되고 해당하는 파일에
content을 작성하면 된다.
Default 설정은 _config.yml을 수정하면 된다.

Content 수정

Content 수정은 단순하다. Content를 수정한 후 새로 포스팅한 것처럼 한다.
혹시 적용이 안될 시 clean 시키고 다시 deploy한다.

Content 삭제

삭제는 직접 수동으로 해야 한다.
포스팅이 저장되어 있는 source_posts폴더에 가서 포스팅을 삭제 하고 클린시키고 다시 디플로이 한다.
